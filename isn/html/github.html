GitHub et le versioning

Présentation et configuration

Je qualifierais GitHub de réseau social orienté vers le partage de code, et la sauvegarde sur un cloud. Mais cette description n'engage que moi !

Github est une plateforme graphique permettant de gérer Git.

  • GitHub Pour les Nuls : Pas de Panique, Lancez-Vous ! (Partie 1) : blog d'aide pour débuter avec GitHub.
  • GitHub pour les Débutants : Consignez, Poussez et Foncez ! (Partie 2) : la suite du blog d'aide pour débuter avec GitHub.
  • GitHub sur OpenClassrooms) : Gérer votre code avec Git et GitHub.
  • Retrouvez-moi sur GitHub : Mon compte GitHub. Très peu d'intérêt pour autrui, ça me sert juste de cloud !
  • Git GitHub et Atom : Utiliser Git dans l'éditeur Atom.
  • Contrôle de Version : Git conserve des "instantanés" de chaque point dans l’historique d’un projet.
  • Commit : C’est la commande qui donne à Git toute sa puissance. Quand vous « committez », vous prenez un "instantané de votre dépôt, vous donnant un point de contrôle que vous pouvez ensuite réévaluer ou restaurer votre projet à un état précédent.
  • Branche : Une branche est une "déviation" dans le projet qui peut se développer de son propre côté pour finalement (ou pas !) être reconnectée à la branche principale du projet.
