Git & GitHub: contrôle de version (versioning)

Présentation

Git est un logiciel de contrôle de version (versioning)

Le principe global est de pouvoir garder une trace de l'évolution d'un projet (ensemble de fichiers) et de pourvoir revenir à toute étape intermédiaire enregistrée (commit).

Il est également possible de développer le projet sur des branches parallèles (branch) à la branche principale (master), et de fusionner différentes branches (merge).

Le projet est sauvegardé localement dans un dossier appelé dépôt (repo ou repository). Il peut être conservé sur un serveur distant (remote).

Pour rapatrier un remote dans un repo local, on peut intialement le clôner (clone) ; ensuite on fait des actualisations avec des pull.

Pour envoyer ses modifications à la suite d'un commit vers le remote, on fait des push.

GitHub est une plateforme en ligne, sorte de réseau social du geek, qui permet de gérer des remote. Il faut créer un compte sur le site de GitHub pour en devenir un utilisateur.
Git peut fonctionner sans GitHub. Mais GitHub est bien pratique ! Il permet de faire pas mal d'actions en mode graphique dans un navigateur.

Vocabulaire

  • Contrôle de Version : Git conserve des "instantanés" de chaque point dans l’historique d’un projet.
  • Commit : C’est la commande qui donne à Git toute sa puissance. Quand vous « committez », vous prenez un "instantané de votre dépôt, vous donnant un point de contrôle que vous pouvez ensuite réévaluer ou restaurer votre projet à un état précédent.
  • Branche : Une branche est une "déviation" dans le projet qui peut se développer de son propre côté pour finalement (ou pas !) être reconnectée à la branche principale du projet.

Mise en place des outils

Création d'un compte GitHub en ligne

Tout d'abord, il faut se créer un compte sur GitHub.

Installation de Git sur votre ordinateur

Ensuite, il faut télécharger et installer Git.

Paramétrage de Git en lien avec le compte GitHub

Une fois que Git est installé, il faut exécuter les deux commandes suivantes :


git config --global user.name "Votre nom ou pseudo"
git config --global user.email "Votre@email.com"

En renseignant évidemment les mêmes identifiants que sur le compte GitHub.
