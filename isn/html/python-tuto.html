  • def fonction(*event) : signature d'une focntion pouvant être apelée à partir d'un gestionnaire d'événement ou de façon "volontaire".

def fonction(*event)

Il arrive qu'on ait besoin qu'une fonction puisse être appelée soit lors de la réaction à un événement de l'utilisateur (clic, appui sur une touche, etc), soit lors d'un appel commandé par un bouton ou par la logique d'une autre fonction du programme.

Le problème est qu'une fonction appelée par le gestionnaire d'événements reçoit obligatoirement comme argument l'événement qui a déclenché son appel, contrairement à un appel "standard". La signature de la fonction doit donc pouvoir s'adapter à ces deux types d'appels.

La solution est simple : utiliser un argument optionnel dans la signature selon le modèle suivant :


def maFonction(*event):
